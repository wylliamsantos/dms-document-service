# Comercial — Handoff Go-to-Market (Alpha)

Este documento consolida o pacote mínimo de go-to-market em um checklist de handoff para operação comercial.

## 1) Pré-requisitos para iniciar venda alpha

- Pricing alpha aprovado (`commercial-pricing-alpha.md`)
- Termos comerciais revisados (`commercial-terms-alpha.md`)
- Política de privacidade revisada (`commercial-privacy-policy-alpha.md`)
- Playbook de suporte publicado (`commercial-support-playbook-alpha.md`)
- Templates operacionais prontos (`commercial-support-templates-alpha.md`)
- Checklist Go/No-Go preenchido (`commercial-go-live-checklist-alpha.md`)

## 2) Critérios de aceite por área

### Comercial
- [ ] Oferta por plano comunicada sem ambiguidade (TRIAL/STARTER/PRO/ENTERPRISE)
- [ ] Regras de upgrade/downgrade alinhadas com billing técnico
- [ ] Mensagem padrão de objeções preparada

### Jurídico/Compliance
- [ ] Termos comerciais aprovados para rodada alpha
- [ ] Política de privacidade aprovada para operação alpha
- [ ] Versões publicadas e referenciáveis para auditoria

### Operações/Suporte
- [ ] SLA de resposta por severidade validado
- [ ] Fluxo de escalonamento P1/P2 testado com simulação
- [ ] Templates de comunicação aprovados

## 3) Evidências mínimas do handoff

Anexar no ticket/PR de go-live:

1. Link do checklist Go/No-Go preenchido
2. Link das políticas e termos vigentes
3. Captura/log de drill de suporte (P1 ou P2)
4. Responsáveis nomeados para comercial, suporte e decisão final de go-live

## 4) RACI mínimo (alpha)

- **Responsible (R):** Operação comercial + suporte
- **Accountable (A):** Dono do produto (go/no-go)
- **Consulted (C):** Engenharia + Jurídico
- **Informed (I):** Stakeholders de negócio

## 5) Condição de saída

O handoff comercial alpha é considerado concluído quando:

- todos os critérios de aceite acima estão marcados;
- evidências mínimas foram anexadas;
- decisão final de Go/No-Go foi registrada com timestamp e responsável.
